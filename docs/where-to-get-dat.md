# Where to get `.dat` files for RADEX
DO **NOT** GO TO EMAA (excitation of molecules and atoms for astrophysics)  
Due to file format issues, using molecular data from EMAA will cause RADEX errors.  
More description about this error can be found in [try_radex-pipeline.md](try_radex-pipeline.md).    

Please use [LAMDA database](https://home.strw.leidenuniv.nl/~moldata/) and download `.dat` file from there :)

The .dat files I found on others' GitHub all have almost the same reference: Yang et al (2010).  
And [LAMDA](https://home.strw.leidenuniv.nl/~moldata/) can link to [RADEX](https://sronpersonalpages.nl/~vdtak/radex/index.shtml), so the format og `.dat` files from [LAMDA](https://home.strw.leidenuniv.nl/~moldata/) should be correct!
